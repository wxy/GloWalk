# GloWalk 产品设计文档

> 中文名：随行路灯 | 英文名：GloWalk | 品牌语：踽踽独行，脚下有光

---

## 1. 产品概述

**GloWalk** 是一款专为 iOS 打造的夜路行走手电筒。不做"万能工具箱"，只做好一件事——让走夜路的人，照明更聪明、路径可回溯、独自更安心。

### 1.1 目标用户与场景

| 维度 | 描述 |
|------|------|
| **用户** | 夜间需要步行的人——加班回家、遛狗、夜跑后 cool down、校园夜归、露营 |
| **场景** | 光线不足的街道/小区/校园/乡间小路，用户需要手机照明看清脚下和前方 |
| **核心 JTBD** | "我不想因为看不清路而摔倒/踩水坑，也不想为了一路照明把手电筒开到没电" |

### 1.2 平台

- **优先**：iOS（本设计文档仅覆盖 iOS）
- **远期考虑**：Android（平台差异大，先做透一个平台）

## 2. 设计原则

> 这些不仅是内部设计原则，也是我们希望用户感受到的产品气质。

### 2.1 夜行不打扰
> *屏幕是暗的，光是暖的。*

UI 采用纯黑底色与琥珀色元素，不破坏暗视力。走路时你不需要"使用"App——它安静地工作，你需要的时候，余光就能扫到关键信息。

### 2.2 抬手即亮，放手即忘
> *一次设置，次次无声守护。*

锁屏小组件、灵动岛、Siri 捷径、背部轻点——多种快捷入口让启动快到来不及数秒。不需要时，它安静退场，不刷存在感。

### 2.3 光有智慧，不只有开关
> *你的每一步，光都懂。*

环境光、屏幕亮度、暗适应时间、月相、天气、姿态——六维感知驱动亮度自适应。不是简单地开灯关灯，而是理解你此刻真正需要多少光。

### 2.4 你的足迹，只留在你的手机里
> *走过的路，只有你知道。*

所有路径数据、光照记录全部本地存储，绝不上传。隐私不是功能，是底线。

## 3. 宣传语

> 每句宣传语遵循"一句话承诺 + 一句解释"的结构。承诺说清价值，解释说清做法。

### 3.1 入口

**快捷启动 —— "抬手即亮，比翻控制中心更快"**
锁屏小组件、灵动岛、Action Button、背部轻点、Siri 捷径。五种方式任你选，最顺手的那种，就是最快的。

**步行界面 —— "屏幕暗下去，世界亮起来"**
纯黑 HUD 搭配琥珀色信息，不刺眼、不费电。余光一扫就知道走到哪了、灯还能亮多久。

### 3.2 智能照明

**环境光自适应 —— "路灯下有分寸，暗巷里不迟疑"**
走到亮处自动调暗省电，走进暗处自动加亮护行。不是粗暴的开关，而是平滑地跟着光线走。

**姿态感应 —— "抬起手机，它知道你要看屏幕"**
竖直看消息时自动熄灭，恢复照明姿态时重新点亮。没有多余的按钮，手怎么放，光就怎么跟。

**瞳孔补偿 —— "屏幕越亮，脚下越需要光"**
屏幕亮度高时瞳孔收缩，你需要更亮的光才能看清路面。而我们的步行 UI 本来就是暗的——帮你保持暗视力，灯光也不用那么亮。

**暗适应曲线 —— "走了五分钟，眼睛已经适应了黑暗"**
人眼在暗处会逐渐适应。走夜路越久，灯光可以越暗。不知不觉间，省下的电够你多走一公里。

**月相感知 —— "满月夜，借一点月光就够了"**
月亮的亮度差可达三百倍。满月之夜，GloWalk 知道不用和你抢那点电；新月之夜，它知道自己必须全力以赴。

**天气感知 —— "下雨天的路面，光照上去是不一样的"**
雨后沥青反光更亮，雪地反射率高达九成。天气不同，光该怎么打也不同。GloWalk 读天气，不是为了气象播报，是为了让你的每一步都踩得踏实。

**遮挡检测 —— "放进口袋的光，是浪费也是隐患"**
闪光灯被遮挡时自动熄灭。不放无用的光，也不让密闭空间里的热量悄悄堆积。

**手动调节 —— "自动的归自动，你的归你"**
觉得自动亮度不对？手指在屏幕边缘轻轻一滑。你的偏好会被记住，但它从不假设你永远这么想。

### 3.3 续航

**电池预估 —— "42 分钟。不是电量，是安心"**
不是冷冰冰的百分比，而是基于当前亮度、步行速度和环境预判的实时估算。"够走到家吗？"——这个数字回答了一切。

**WiFi 到达感知 —— "连上家里的 WiFi，光就知道到家了"**
不用设置目的地。检测到熟悉的 WiFi，GloWalk 就知道你已经安全到达。不用记得关，它帮你记得。

### 3.4 足迹

**路径记录 —— "走过的夜路，会变成一张地图"**
记录你的 GPS 轨迹和每一步的光照。数据只在你的手机里，生成只属于你的夜行档案。

**路径海报 —— "每一段夜路，都是一幅画"**
用卡通笔触画出你的夜路足迹。明暗线条标记光亮变化，最黑的那段路，是你征服过的证明。可以分享，也可以只留给自己。

### 3.5 安全

**安全签到 —— "走完这段夜路，让关心你的人知道"**
到达后生成一张海报，分享给朋友。一张图比一句"我到了"更温暖。连上家里的 WiFi，GloWalk 就知道你已经安全到家。

### 3.6 同行（V2 预留）

**近场发现 —— "附近也有 GloWalk 的路人，要一起亮吗？"**
无需配对、无需蓝牙。手机自动感知周围的同行者，你们的光可以连成一片。

**流水灯 —— "三个人一排，就是一串行走的路灯"**
自动识别排列顺序，灯光按着队形脉动流转。一个人照亮脚下，一群人照亮彼此。

### 4.5 数据模型（Core Data 实体草案）

```
WalkSession（步行会话）
  ├── id: UUID
  ├── startTime: Date
  ├── endTime: Date?
  ├── totalSteps: Int
  ├── totalDistance: Double         // 米
  ├── avgLightLevel: Double         // 平均环境光照度
  ├── moonPhase: String
  ├── weatherCondition: String?     // iOS 15 = nil
  └── pathPoints: [PathPoint]       // 一对多

PathPoint（轨迹点）
  ├── latitude: Double
  ├── longitude: Double
  ├── timestamp: Date
  ├── ambientLight: Double          // 环境光照度
  └── torchBrightness: Double       // 该点的手电筒亮度
```

### 4.6 融合决策器算法

```
最终亮度 = (E × wₑ + P × wₚ + S × wₛ + M × wₘ + W × wᵥ + D × wₔ) / 总权重
                                    + 用户手动偏移量

其中：
  E = 环境光信号（暗 → 建议高亮度）
  P = 姿态信号（非照明姿态 → 建议 0 或低亮度）
  S = 屏幕亮度补偿（屏幕亮 → 建议偏高亮度）
  M = 月相信号（满月 → 建议偏低亮度）
  W = 天气信号（雨雪反光 → 建议偏低亮度）
  D = 暗适应信号（走暗路越久 → 建议逐步降低亮度）
```

**各信号权重（初始值，需实测调优）：**

| 信号 | 权重 | 特性 |
|------|:----:|------|
| 环境光 (E) | **40%** | 主导信号，光线环境决定基调 |
| 姿态 (P) | **25%** | 高权重——非照明姿态直接干预 |
| 屏幕亮度 (S) | **10%** | 辅助补偿，影响中等 |
| 暗适应 (D) | **10%** | 缓慢作用，润物细无声 |
| 月相 (M) | **10%** | 满月/新月差异大，但仍是辅助 |
| 天气 (W) | **5%** | 最弱的信号，仅做微调 |

**滞后逻辑（防闪烁）：**

- 关闭 → 开启：环境光 < 8 lux，且持续 2 秒以上
- 开启 → 关闭：环境光 > 35 lux，且持续 3 秒以上
- 亮度调整平滑过渡：100ms/step，避免跳变

**遮挡检测：**

- 每 5 秒采集一次后摄帧（64×48 像素，取完即弃）
- 画面平均亮度 > 200/255 且纹理方差低于阈值 → 判定遮挡 → 0.5s 渐暗至关闭 + 轻振动
- 画面平均亮度 < 10/255 且纹理方差低于阈值 → 照向空旷天空，维持当前亮度（合理场景不干预）

**姿态判定：**

- 有效照明姿态：pitch 30°-60°，roll -15° 到 15°
- 非照明姿态需保持超过 2 秒才触发（"意愿窗口"，避免误触发）
- 竖直（pitch > 70°）：看屏幕 → 渐暗
- 下垂（pitch < 10°）：自然下垂 → 渐暗
- 横滚过大（|roll| > 45°）：横置手机 → 渐暗

### 4.7 前台/后台照明策略

iOS 的核心限制：锁屏或 App 进入后台后，AVCaptureSession 被系统中断，所有传感器采样停止——环境光、姿态、遮挡检测全部失效。但闪光灯硬件级别的 torchMode 会保持。

**因此灯会继续亮，但不再"聪明"。**

对应策略——两级照明模式：

```
┌──────────────────────────────────────────────┐
│              前台 · 智能模式                   │
│                                              │
│  全传感器激活                                  │
│  六维融合决策（环境光 + 姿态 + 屏幕 + 暗适应    │
│    + 月相 + 天气）→ 实时自适应亮度              │
│  遮挡检测活跃                                  │
│  亮度范围：10% - 100%                         │
│  UI：HUD 完整展示所有决策因子                   │
└────────────────────┬─────────────────────────┘
                     │
          ┌──────────▼──────────┐
          │ 检测到即将进入后台    │
          │ (applicationWill    │
          │  ResignActive)      │
          └──────────┬──────────┘
                     │
                     ▼
┌──────────────────────────────────────────────┐
│           后台/锁屏 · 安全兜底模式              │
│                                              │
│  立即执行：setTorchMode(1.0)  ← 拉满亮度      │
│  所有传感器休眠                                │
│  闪光灯固定在最高亮度                           │
│  无自动调节                                    │
│  iOS 16+：Live Activity 显示状态 + 手动调节    │
│  iOS 15：仅本地通知，无锁屏交互                │
└──────────────────────────────────────────────┘
```

**设计理由**：

> 当 App 无法继续感知环境时，我们不知道用户走进了暗巷还是站在路灯下。此时"最亮"是唯一安全的默认值——宁可让用户觉得有点亮、手动调暗，也不能让用户在暗处因为没有足够的光而绊倒。

用户解锁回到前台时：

```
  安全兜底模式（100% 固定亮度）
         │
         │  用户解锁，App 回到前台
         │  applicationDidBecomeActive
         ▼
  ┌─────────────────────────────┐
  │  传感器重新激活              │
  │  融合决策器重新计算目标亮度   │
  │  例如：环境光 + 姿态 + ...  │
  │       → 目标亮度 35%        │
  └─────────────┬───────────────┘
                │
                ▼
  ┌─────────────────────────────┐
  │  平滑过渡                    │
  │  100% → 35%                │
  │  以 10%/step 的速度渐变     │
  │  耗时约 1-1.5 秒            │
  │  用户感知："灯光自动柔和下来" │
  └─────────────┬───────────────┘
                │
                ▼
        智能模式（自适应亮度）
```

过渡期间不触发滞后逻辑（避免传感器刚激活时的初始值波动导致二次跳变）。过渡完成后再开启正常的自适应循环。

- 闪屏期间手电筒未开启，用户处于正常光照环境
- 宣传语从预设库中随机选取（每日轮换或每次启动轮换）
- 过渡动画：屏幕亮度在 1 秒内降至最低，白底渐变为纯黑，宣传语淡出，发光圆环淡入
- 过渡完成后手电筒自动开启，进入智能模式


**月相/天气信息卡**（位于 HUD 底部，信息栏上方）：

每张卡约 12-14pt 字号，两行紧凑排列，琥珀色半透明：

```
  🌙 满月                   ☁️ 小雨
  亮度 -15%                反光补偿 +10%
```

| 属性 | 说明 |
|------|------|
| 内容 | 图标 + 当前状态 + 对亮度的实际影响（百分比） |
| 交互 | **可点击切换**。点击后卡片变灰 + 删除线，对应因子暂停，仅对当前步行有效 |
| 默认 | 每次步行自动开启 |
| 无数据时 | ☁️ 卡不显示（iOS 15 或无网络）；🌙 始终显示（离线天文计算） |

**为什么只有这两个？**

- 🌙 和 ☁️ 是唯一两个"用户可能在某次步行中不同意自动决策"的因子——满月夜想亮一点、下雨天不想让天气影响亮度。这是单次步行的临时覆盖。
- 姿态感应和暗适应是系统默认行为，不可操作也不需要展示——姿态是即时反馈（用户举手机灯就变），暗适应本就不该被感知到。
- 环境光和遮挡检测更属于后台基础设施，展示出来纯属噪音。

**手势操作：**

| 手势 | 动作 | 说明 |
|------|------|------|
| 屏幕右侧边缘单指上滑 | 亮度 +10% | 盲操作 |
| 屏幕右侧边缘单指下滑 | 亮度 -10% | 盲操作 |
| 屏幕边缘双指长按 | 恢复自动亮度 | 取消手动偏移 |
| 双击屏幕任意位置 | 关闭手电筒 | 最快速的关闭方式 |

手势设计目的：用户走路时不需要低头找按钮，手指摸到屏幕边缘就能盲操作。

### 5.6 步行结束

步行结束有两种路径，处理方式不同：

**路径 A：用户主动点击"结束并通知"按钮**

```
  HUD 底部 [结束并通知] 按钮
       │
       ▼
  手电筒平滑关闭（0.5s 渐暗）
       │
       ▼
  自动生成海报（AI 管线或本地降级）
       │
       ▼
  ┌─────────────────────────────────┐
  │  🎉 安全到达！                  │  ← 亮屏海报预览
  │                                 │
  │  ┌─────────────────────────┐    │
  │  │                         │    │
  │  │     [海报预览图片]       │    │  ← 自动生成的海报
  │  │                         │    │
  │  └─────────────────────────┘    │
  │                                 │
  │  🚶 1.8km  ·  暗光路段 1.2km   │
  │  🌕 满月夜                     │
  │                                 │
  │  "走完这段夜路，让关心你的人知道"│  ← 宣传语 + 解释
  │                                 │
  │  [分享]  [查看历史]  [完成]     │
  └─────────────────────────────────┘
```

- App 回到正常屏幕亮度 → 自动生成海报 → 展示预览 + 分享按钮
- 场景类比：朋友送用户到某地后，用户走完最后一段夜路，到达后分享海报告知安全抵达
- 海报生成时间可控（AI 管线优化目标 < 3 秒），若超时则先展示摘要，海报后台生成后补推通知

**路径 B：用户强制关闭 App（上滑杀进程）**

```
  用户上滑杀 App
       │
       ▼
  applicationWillTerminate 触发
       │
       ▼
  ┌─────────────────────────────┐
  │  清理动作（尽力而为）：       │
  │  1. 关闭闪光灯              │
  │  2. 保存当前 WalkSession    │
  │  3. 标记为"中断结束"        │
  └─────────────────────────────┘
  
  无法展示摘要（App 正在被杀死）
  下次打开 App 时，首页轻提示：
  "上次步行未正常结束，已自动保存记录"
```

> **注意**：`applicationWillTerminate` 不保证一定会被调用（系统可能在极端情况下直接杀死进程）。因此闪光灯的关闭需要在更早的时刻就已经做好准备——实际上，当 App 进入后台时（4.7 节），手电筒已固定在安全兜底模式，而如果用户在后台直接杀进程，系统会在进程终止时自动释放闪光灯硬件锁，灯会灭。这是 iOS 的系统级保底。

### 5.7 设置页面

#### 设计原则：能默认的就不让用户选

GloWalk 的智能照明不是一个"功能开关列表"，而是一个完整的体验。绝大多数因子（环境光、姿态、暗适应、屏幕亮度补偿、遮挡检测、滞后逻辑、平滑过渡）是**系统默认行为**，不出现在设置中。如果让用户逐个关闭，App 就退化成了普通手电筒。

用户可配置的是两类东西：**个人偏好**和**需要用户信息的可选功能**。

| **已拒绝** | [前往系统设置] + [请求授权] | [请求] → 重新触发系统授权弹窗 |
| **未询问** | [前往系统设置] + [请求授权] | [请求] → 触发系统授权弹窗 |
| **设备不支持** | 灰色文字"此设备不支持" | — |

**设计理由**：

- 让用户在一个页面看到"授予了什么"和"获得了什么"的全景，建立信任
- 已授权状态下用 ✓ 标记并列出启用的子功能——让用户觉得授权是值得的
- 已拒绝状态下用户看到"失去的功能"列表，可能因此改变主意
- 数据清除放在同一页面，强化"所有数据你都能控制"的信息

## 6. 路径记录与海报生成

### 6.1 路径记录策略

| 参数 | 建议值 | 说明 |
|------|--------|------|
| GPS 采样间隔 | 每 10 秒一次，或每 10 米一次 | 平衡精度与功耗 |
| 光照度采样 | 与 GPS 同步记录 | 每个轨迹点附带当时的环境光读数 |
| 停止记录条件 | 手电筒关闭 或 用户手动结束 | 自动开始，自动结束 |

**数据量估算**（一次 30 分钟步行）：
- 180 个 PathPoint × ~100 bytes ≈ **18 KB**
- 全年每天一次 ≈ **6.5 MB**，本地存储零压力

### 6.2 海报生成

#### 视觉方向

最终定位于 **卡通风格**，非写实地图。用户分享到社交媒体后，相当于免费传播，因此可以在海报生成上投入一定成本。

#### 方案对比

| 方案 | 成本 | 效果 | 开发量 | 
|------|:---:|:----:|:-----:|
| **A. 纯本地滤镜** | 🟢 零 | 🟡 有限的卡通化（色块量化 + 边缘强化 + 暖色滤镜），更像是"滤镜处理过的地图"而非真正的卡通 | 低 |
| **B. Mapbox 自定义风格** | 🟡 免费额度内零成本 | 🟡 可自定义地图配色和线条风格，偏扁平插画风，但仍可辨认出是地图 | 中 |
| **C. AI 图像生成** | 🔴 每次 ~$0.02-0.10 | 🟢 真正的卡通/手绘风格，可融入多种艺术效果 | 中 |
| **D. 混合方案（推荐）** | 🟡 低成本 | 🟢 兼具可控性与表现力 | 中 |

#### 推荐：混合方案 D

```
阶段 1（本地免费）：MKMapSnapshotter 快照
  → Core Image 色块量化（减少颜色数 → 色块感）
  → 边缘检测叠加（模拟手绘线条）
  → 暖色映射（深色 → 深蓝/紫，亮色 → 琥珀/暖黄）
  
  效果：类似 2D 平台游戏地图，扁平色块 + 线条感

阶段 2（本地免费）：非地图元素叠加
  → 路径线用粗细+颜色表示光照（Core Graphics）
  → 标注点用对话气泡
  → 月相图标、天气图标
  → 底部文字排版

阶段 3（AI 增强，可选）：将阶段 2 的输出作为输入图
  → 发送到 AI 图像 API 做风格迁移
  → prompt: "a cozy night walk path illustration, 
     hand-drawn cartoon style, warm amber street lights, 
     dark blue night sky, soft watercolor texture"
  → AI 保留路径结构和文字，增强艺术感
  → 成本：~$0.03/张，受每日 1 张额度限制
```

#### AI 服务选型

| 服务 | 模型 | 单张成本 | 风格质量 | 备注 |
|------|------|:-------:|:--------:|------|
| **OpenAI** | GPT-image-1 / DALL-E 3 | ~$0.04-0.08 | 🟢 优秀 | 生态成熟，图生图能力强 |
| **Stability AI** | Stable Diffusion | ~$0.01-0.03 | 🟡 取决于 prompt | 成本最低，需调参 |
| **Replicate** | 各种开源模型 | ~$0.005-0.02 | 🟡 参差不齐 | 模型多，适合实验 |

> 由于 App 定位为**付费应用**，AI 生成成本可以内化在购买价格中。采用每日 1 张的额度策略，活跃用户每月最多 30 张，成本约 $0.90/月，作为获客和传播成本极低。不限总生成数量的策略使得每张海报都是潜在的社交传播素材——用户分享得越多，App 的自然增长越强。

#### 不去写实地图的技术原因

- `MKMapSnapshotter` 输出的快照是 Apple Maps 的标准视觉风格，无法替换地图图块
- 将其彻底"卡通化"需要逐像素后处理，纯 Core Image 效果有限
- 如果用户期望的是真正的卡通/手绘风格，AI 是最直接的路径

## 7. 安全到达

### 7.1 设计原则

不做复杂的紧急联系人系统。只做一件事：**让用户到达目的地后，轻松告知关心的人"我到了"**。通过一张海报代替一句文字。

### 7.2 WiFi 自动到达检测（推荐方式）

大多数夜间步行的终点是固定建筑——家或公司。这些地方有已知的 WiFi。

```
用户预先配置（设置 → 安全到达）：
  "到达 WiFi" → 选择已保存的网络
  例如：🏠 家 → "MyHome_5G"
        🏢 公司 → "Office_WiFi"

步行中：
  后台监测 WiFi 连接变化（NEHotspotHelper / 系统通知）
  
  检测到已配置的 WiFi 连接：
    → 弹窗："检测到你已连上 MyHome_5G，是否已到家？"
    → [已到家 ✓] → 自动生成海报 → 分享
    → [还没到]   → 继续监测
```

- 比 GPS 地理围栏更可靠：室内 GPS 漂移严重，WiFi 是确定性信号
- 零额外耗电：不需要持续 GPS 采样，仅监听系统 WiFi 事件
- 隐私友好：仅匹配 SSID，不需要知道 WiFi 密码或网络内容

### 7.3 手动设置目的地（备选方式）

对于非常规目的地（如朋友家、酒店），用户可在出发前手动选点：

- 从地图选点，或从步行历史中复用
- 可选输入备注："去小明家"
- 到达后同样自动生成海报 → 分享

### 7.4 结束并分享

无论以哪种方式到达，最终体验一致：

```
用户确认到达
     │
     ▼
自动生成海报（含品牌水印 + 随机宣传语 + 解释）
     │
     ▼
展示海报 + 分享按钮
  [分享到微信]  [分享到 Messages]  [保存到相册]  [关闭]
```

一张海报胜过一句"我到家了"——它既报了平安，也完成了品牌传播。

> **刻意不做的事**：紧急联系人自动通知、超时未到达报警、GPS 实时位置共享。这些功能让 App 变得沉重，且 iOS 的平台限制使它们实际上不可靠。GloWalk 只做到"帮用户优雅地报平安"为止。

预设轮换池（短语 + 解释）：

| 短语 | 解释 |
|------|------|
| "路灯下有分寸，暗巷里不迟疑" | 摄像头每 5 秒采样环境光，滞后逻辑防止灯光在临界值反复跳变 |
| "走过五分钟，眼睛已经适应了黑暗" | 暗适应曲线让灯光随步行时长逐步调暗，不知不觉间省下电量 |

轻点 Tips 可跳转到设置中对应的功能开关。

### 8.5 海报生成策略

**不限制总量，限制每日次数。**

| 规则 | 说明 |
|------|------|
| **自动生成** | 每次手动"结束并通知"时自动触发海报生成，消耗当天额度 |
| 每日生成额度 | **1 张/天**（本地计数，不依赖服务器） |
| 可补生成 | 可为任意历史日期的步行记录生成海报，消耗**当天的额度** |
| 当天步行 | 若当天额度已用尽，手动结束时仍生成摘要卡片，但不触发 AI 海报 |
| 额度重置 | 每日 0:00 本地时间重置 |

**自动生成流程**：

```
用户点击"结束并通知"
       │
       ▼
  检查当天额度
   ├── 有额度 → AI 海报生成（目标 < 3 秒）
   │              └── 成功 → 展示海报 + 分享按钮
   │              └── 超时 → 先展示摘要卡片
   │                        海报后台生成 → 完成后推送通知
   └── 无额度 → 展示摘要卡片（不含海报）
                底部提示："今日海报额度已用完，明天再来"
```

**设计逻辑**：

- 每张海报都是用户的社交分享素材 → 等于**免费广告**
- 每天 1 张制造稀缺感，用户会更珍惜每次分享机会
- 允许补生成让用户不会因为"今天没走夜路"而浪费额度
- 用户平均每周分享 1-2 次已是极高的传播频率
- AI 成本可控：活跃用户每月 ~30 张 ≈ $0.90，仍远低于一次购买收入

**未来扩展（V2）**：基于历史记录生成汇总海报——"本周夜行报告"、"本月步行地图"、"年度夜路足迹总览"。
